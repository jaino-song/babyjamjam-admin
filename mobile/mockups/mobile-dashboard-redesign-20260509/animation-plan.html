Mobile UI Animation Plan — 시각 미리보기

예전 UI에서 잃었던 자연스러운 motion을 복원하되, 과하지 않게 적용하는 플랜. 모든 예시는 플랜의 모션 토큰(duration·easing)을 그대로 사용해서 구현 후 느낌과 동일합니다. 하단의 버튼으로 직접 재생/비교해보세요.

📌 Context — 왜 이 작업이 필요한가

예전 mockup (mockups/mobile-dashboard-redesign-20260509/)에는 의도된 motion 언어가 있었습니다 — iOS-style 페이지 전환, 절제된 카드 entrance, 명확한 timing system. 현재 redesign은 mockup을 시각적으로 가져왔지만 일부 animation이 사라졌습니다: 바텀 nav의 active indicator는 instant, contracts/messages/files 리스트는 한꺼번에 등장, .badge 색 전환은 instant, detail sheet 안 tab content swap도 snap. prefers-reduced-motion 커버리지도 부분적.

좋은 소식: 인프라는 이미 갖춰져 있습니다 — framer-motion@12.23.24 설치됨, globals.css에 25+ keyframe, redesign.css에 spatial nav transition. 부족한 건 capability가 아니라 application + 절제. 이 플랜은 새 라이브러리·새 추상화 없이 기존 utility를 빠진 자리에 적용합니다.

🗺️ Architecture — 두 레이어, 각 레이어의 owner를 정확히 타게팅

이 코드베이스에는 두 motion 아키텍처가 공존합니다. Plan은 각 레이어에서 올바른 file을 수정합니다.

SECONDARY v3/

Foundation

Motion Design Tokens

기존 25+ keyframe들은 그대로 두고, 의도 기반(semantic) 토큰만 새로 정의합니다. duration은 "얼마나 빠르냐"가 아니라 "어떤 intent의 모션이냐"로 명명. 이렇게 하면 junior dev나 AI agent도 mechanical 하게 올바른 토큰을 선택합니다.

1.6 — 루트 template.tsx는 이번 롤아웃에서 제외 body-class shell-reshape 충돌

⚠️ Risks & Mitigations

각 변화에서 잘못 갈 수 있는 지점과 미리 막아두는 방법.

RiskMitigation
Stagger가 async 리스트 로딩 시 layout shift 유발 animation: ... both로 시작 state hold. placeholder 클래스에는 animation 미적용. ListItemRow가 실제 데이터를 받았을 때만 발화.
React re-render마다 animation 재생 CSS keyframe + animation-fill-mode: both (globals.css:617의 v3-slide-up-children이 이미 그렇게 동작). React re-render는 element가 unmount/mount되지 않는 한 CSS animation을 재시작하지 않음.
Low-end Android 성능 모든 새 animation은 transform·opacity만 사용 (GPU). will-change는 기존 위치만 유지, 새로 추가 X (남용 시 오히려 해로움).
Badge crossfade가 행 unmount 시 flicker Crossfade는 badge 클래스 변경 시에만 발화. 행이 remount되면 행의 entrance animation이 담당 (중복 X).
DetailTabPills tab content가 두 개 잠시 겹침 AnimatePresence mode="wait"로 old exit 완료 후 new mount 보장.
body class 변경이 route motion과 race Phase 1.6에서 루트 template.tsx 명시적으로 제외. 기존 .nav-stack 모션으로 충분.

✅ Verification — 어떻게 검증하나

각 phase 종료 시 ship gate를 통과해야 다음으로. 명령은 모두 package.json에 실제 존재.

  • Type check package.json에 type-check script가 없음 → npx tsc --noEmit 실행 (next build의 묵시적 check도 OK).
  • Lint · Test npm run lint, npm run test, 필요 시 npm run test:screenshots.
  • iOS Safari · Android Chrome 비디오 바텀 nav 탭 전환, 리스트→detail 슬라이드, 탭 content 크로스페이드, 뱃지 상태 변경, 폼 validation 등록 시 시각 비교.
  • Reduced-motion Chrome DevTools → Rendering → "Emulate prefers-reduced-motion: reduce" → 모든 animation 사실상 사라지지만 layout 유지, spinner는 1s로 계속 돌아감, pulse는 opacity 0.6에 고정.
  • Performance throttling Chrome DevTools Performance, 4× CPU throttle, 200+ 행 리스트 스크롤 → frame drop 없어야 함.
  • Playwright regression tests/dashboard-activities-animation.spec.ts는 dashboard activities 패널의 v3-* 이벤트만 커버. Phase 1·3 후 regression check (새 surface의 positive coverage는 follow-up 작업).
