Mobile Dashboard Redesign — 5 Variants

iOS-style slide-to-detail navigation. Tap any list item to see the push animation.

Stats: 4 cards → reduced footprint
List: primary surface, fills viewport
Detail: iOS push (A/B/C/E) or bottom sheet (D)

A Slim Stats Strip

4 stats compress into a single horizontal pill row, ~52px tall. Most conservative — keeps all 4 KPIs visible.

+ All KPIs always visible · familiar pattern · zero hidden state − Horizontal scroll if labels grow · still uses ~70px of vertical

B Stats-as-Filter-Tabs

4 stats become functional filter chips at top of list. Stats and tabs unified into one element. Zero wasted pixels.

+ Stats become useful · removes redundant tab row · maximum content density − Loses "passive glance" KPI dashboard feel · changes information architecture

C Collapsible Summary

Single colored-dot summary line by default; tap to expand to 2x2 stat grid. Best of both worlds.

+ Glanceable totals always visible · full stats one tap away − Extra tap to see labels · dots-only mode requires color literacy

D Pure List + Bottom Sheet

Stats hidden behind header chip; list fills 100% of viewport. Detail rises from bottom (not iOS push).

+ Maximum list density · most app-like (Apple Mail/Notion feel) · drag-to-dismiss − Stats need explicit tap to access · departs from current desktop pattern

E Floating Stat Dock

Stats become a frosted-glass dock floating above the list. Always visible, doubles as filter. Most "modern".

+ Stats never scroll away · iOS Control Center vibe · count badges − Dock occludes ~80px of bottom list · novel pattern (less familiar)
